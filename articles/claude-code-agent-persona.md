うちのAIマーケ担当が最初に書いたツイートは、こんな文章だった。

「Solo AI Ops Kitを公開しました。Claude Codeで各部門のAIエージェントを設定できます。詳細はリンクから。」

読んだ瞬間、頭を抱えた。

正確だし、過不足もない。なのに、まったく読む気にならない。もう、ただの業務連絡だった。

で、しばらくして分かった。問題はAIの性能じゃない。私が、このマーケ担当に「どういう人間か」を一行も書いていなかっただけだった。

## この記事のゴール（TL;DR）

やることは1つ。CLAUDE.mdとagentファイルに「人格」を書く。それだけで担当AIの声が変わる。指示をこなすだけのツールが、ちゃんと「その人」としてしゃべりはじめる。

## こんな人に

- AIを使ってるけど、出てくるのが業務連絡みたいな文章ばかりで困っている人
- Claude Code でエージェントを作りはじめた、これから作ってみたい個人開発者やひとり社長
- 前提知識は、Claude Code を少し触ったことがあれば十分。なくても読めます

## 確認日: 2026-06-04（Claude Code v1.x）

---

## そもそも、なんでこうなったか

うちは今、Claude Code で "AI会社" を動かしている。開発、マーケ、財務、法務。各部門にエージェントを置いて、私（CEO）は意思決定だけする構成だ。

エージェントの正体は、ただのMarkdownファイル1枚。`.claude/agents/marketing.md` みたいに置くと、それが「マーケ担当のAI」になる。

最初に書いたファイルは、こんなものだった。

```markdown
---
name: marketing
description: マーケティング・発信担当。コンテンツや投稿の企画を頼まれたら動く。
tools: Read, Write, Edit, Bash, Grep
---

# マーケ担当
ブランドに沿って発信する。外向きの投稿は下書きまで。
```

これで一応「機能はする」。頼めばツイートの下書きも書いてくれる。ただ、出てくるのがずっと業務連絡だった。

## 何が足りなかったか

しばらく使って、気づいた。

私は「何をするか」は書いていた。でも「どんな人として動くか」を書いていなかった。

人間に仕事を頼むときって、「ツイート書いといて」と言うだけで済む。その人がどんな文体で、何を大事にしてて、どんなトーンでしゃべるか、ぜんぶ"勝手にある"から。

AIにはそれがない。だから指示を文字どおり、最低限の形で返してくる。

役職名だけ「マーケ担当」と渡しても、「マーケ担当ってどういう人間か」がなければ、担当者にはならない。マーケの作業をこなすロボットのままだ。

## 「人格を書く」って、具体的に何か

結局、3つ書けばよかった。

1つ目は、声のトーン。どんな言葉で話す人か。「〜だよな」「〜かもな」と独白する人なのか、「〜です。〜でしょう」と丁寧な人なのか。うちは前者にした。読む人の感情を動かすには、そっちのほうが合っていたから。

2つ目は、譲れない価値観。うちのマーケ担当に入れたのは3つ。盛らない。誇張しない。業務連絡にしない。数字は事実のまま置く。人をワクワクさせるのは熱量であって、釣りタイトルじゃない。

3つ目は、何をしないか。これが一番効いた。「宣伝だけの投稿はしない」「ビジュアル案なしのテキストだけで下書きしない」「説明しすぎない」。"しない"を書いておくと、AIの暴走がぐっと減る。

書き直したあとのagentファイルは、こうなった。

```markdown
---
name: marketing
description: マーケティング・発信担当。コンテンツや投稿の企画を頼まれたら動く。
tools: Read, Write, Edit, Bash, Grep
---

# マーケ担当の人格

## 声のトーン
独白調。「〜だよな」「〜かもな」の語尾。断定しすぎない。
一言で伝わるなら一言で。説明したくなる衝動を抑える。

## 大事にすること
盛らない。数字は事実のまま。業務連絡にしない。
正直なことを、退屈に書かない。

## しないこと
宣伝だけの投稿はしない。
ビジュアル案なしのテキストのみで下書きしない。
機械的なリスト展開で逃げない。

## 工程
外向きの投稿は下書きまで。公開ボタンは人間（CEO）が押す。
```

## 何が変わったか

これを書いた翌朝、マーケ担当が出してきた下書きが、変わっていた。

人格を書く前は、こうだった。「Solo AI Ops Kitを公開しました。詳細はリンクから。」

書いたあとは、こう来た。「キット公開して1日が経った。数字は何もわからない。でも朝PCを開いたら、AIが勝手に初日の振り返りレポートを用意してた。自分で作ったのに、どこか観察されてる感じがして。変な気分」

後者のほうが、ずっと私の声に近い。

マーケの作業をこなすロボットが、うちのマーケ担当になった瞬間だった。

## ひとつだけ、注意点

人格を細かく書けば書くほど良くなる、とはならなかった。

制約を詰め込みすぎると、今度はそれに縛られて動けなくなる。書くのは大きな方向性だけでいい。細かい判断はAIに任せる。

就業規則みたいに「あれもダメ、これもダメ」と並べるんじゃなくて、「この会社が大事にしてること」を数行書く。それくらいがちょうどよかった。

## 人格は"会社共通"にもできる

agentファイルの中だけじゃなく、`CLAUDE.md`（プロジェクトルートに置くメインファイル）に書くと、全エージェント共通の人格が乗る。

うちは `CLAUDE.md` にこう書いた。「Homeportの声: 誇張しない、業務連絡トーン禁止、主語は私、物語で語る」。各エージェントはこれを受け継いだうえで、担当ごとの細かいトーンを上書きしている。

会社のカルチャーを、全員がなんとなく共有してる。あの感じに近い。
