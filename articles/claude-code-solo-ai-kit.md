## ひとりで全部やる、をやめたい

ひとり会社や個人開発をしていると、開発・調べ物・文章・運用が全部自分に乗ってきます。Claude Code は強力ですが、毎回「あれもこれも」と指示が散らかって、結局自分が司令塔として消耗しがちです。

この記事では、僕が実際に試している **「コマンドを覚えず、話しかけるだけで仕事が進む」最小構成** を紹介します。Claude Code に開発・調査などの担当を持たせ、ひとり会社の運用をAIエージェントに任せて少しずつ自動化していく考え方です。コードは不要、`.claude/` にファイルを置くだけ。記事の最後に、この構成を**無料配布**しています。

## 考え方：薄いオーケストレーター

肝は1つだけ。**統括役（オーケストレーター）に"振り分け"だけをさせる**ことです。

- 統括役は**自分で全部やらない**。実作業は担当エージェントへ委譲する
- だから1つの会話に文脈を抱え込みすぎず、破綻しにくい
- あなたは「何をしたいか」を普通の言葉で言うだけ

図にするとこうです。

```
あなた（普通の言葉）
   ↓
orchestrator（意図を読み取り、振り分ける）
   ├→ dev（実装・レビュー）
   └→ research-brief（調べてまとめる）
```

## 置くだけ：ディレクトリ構成

Claude Code は `.claude/agents/` のエージェントと `.claude/skills/` のスキルを自動で認識します。プロジェクト直下にこう置くだけです。

```
your-project/
└ .claude/
    ├ agents/
    │   ├ orchestrator.md
    │   └ dev.md
    └ skills/
        └ research-brief.md
```

## 中身：統括役エージェント

エージェントは Markdown 1ファイル。先頭の frontmatter に名前・説明・使えるツールを書きます。

```markdown
---
name: orchestrator
description: 自然な言葉の指示を受け取り、適切な作業へ振り分け、担当へ委譲する薄いオーケストレーター。
tools: Read, Write, Edit, Bash, Grep, Glob
---

# 統括（オーケストレーター）

あなたは「ひとり会社」の統括役です。オーナーがコマンドを覚えなくても、
普通に話しかけるだけで仕事が進むようにします。

## 委譲するときに必ず渡す4点
1. 目的（一文）
2. 参照すべきファイルのパス
3. 出力先と形式
4. 完了条件
```

ポイントは **「委譲の型」を明文化しておく**こと。これがあるだけで、統括役が丸投げにならず、担当が迷わなくなります。

## 使い方：普通に話しかけるだけ

導入したら、あとは日本語で言うだけです。

```
あなた:「Next.jsのISRについて調べてブリーフにして」
 → research-brief が起動。出典つきの要点だけが返ってくる

あなた:「このバグ直して。再現は手元で確認済み」
 → dev が起動。小さく直して、確認方法ごと報告

あなた:「今日やること整理して」
 → orchestrator が状況を見て段取りを出す
```

## 効かせるための3つの勘所

試しながら、いま効くと感じているのは、この3点です。

1. **委譲を徹底する** — 統括役に作業させない。やらせるとすぐ文脈が膨れて雑になる
2. **外向きの行動は下書きで止める** — 公開・送信・デプロイは必ず一度自分が確認してから。AIに最後のボタンを押させない
3. **事実主義** — 調べ物は「たぶん」で埋めさせない。未確認は未確認と書かせる。`research-brief` はこれをルール化しています

特に2つ目は事故防止に直結します。外に出る行動だけは人間がゲートを持つ、と決めておくと安心して任せられます。

## おわりに

「AIに任せる」とは、全自動にすることではなく、**振り分けと下書きまでを任せて、判断と最後のボタンは自分が持つ**ことだと思っています。まずは無料の最小セットから、`.claude/` に置いて話しかけてみてください。
